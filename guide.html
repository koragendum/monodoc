# Introduction

A markdown page or fragment is first broken into block-level elements which are then individually rendered and then concatenated.

You can mix markdown and html, since the markdown parser preserves html tags.

Monodoc uses U+F000 through U+F009 internally to mark locations as it manipulates text, so unfortunately your document can’t use these scalar values.

# Block-level Elements

# Headings

Lines that start with one or more # followed by a space are headings. The number of # indicates the heading level.

# Paragraphs

Anything that’s not one of the other block-level elements is a paragraph. A paragraph is ended by an empty line, the start of another block-level element, or (if you’re mixing markdown and html) one of the following html tags:

h1 h2 h3 h4 h5 h6 div p pre ol ul table

Within paragraphs (but not the other block-level elements) you can indent lines using |n>, where n indicates how far to indent the line.

For any line in a paragraph,
|1> if the line begins with `|1\>` it’s indented once;
  |2> if `|2\>` then it’s indented twice
|2> (regardless of the indentation in the source);
|0> and so on for `|`@n`\>` from @n = 0 up to @n = 9.
For any line, if the line begins with |1> it’s indented once; if |2> then it’s indented twice (regardless of the indentation in the source); and so on for |n> from n = 0 up to n = 9.

# Lists

Lines that start with *, -, or + followed by a space are list items. When a list is invalid (because of mismatching bullets or mismatched indentation), the list is printed as a code block.

# Tables

Lines that start with | are table rows, and | is used to further separate cells in the row. Tables can optionally have a row that indicates column alignment, and if they have an alignment row, they can also optionally have a header. When a table is invalid (because the rows don’t have the same number of columns or there are multiple alignment or header rows), the table is printed as a code block.

# Inline Elements

These can be used in every block-level element besides code blocks. Each heading, paragraph, list item, and table cell is rendered separately.

# Escape Sequences

If you need a literal \ < > & # ` _ * in a place where it’d be interpreted as markdown or html syntax, you can escape the character by typing a backslash first: \\ \< \> \& \# \` \_ \*

(You could also write &lt; or &#x3C; instead of \<. The backslash sequences are provided for convenience.)

# Inline Code

Text between pairs of ` is rendered as inline code.

Inline code is set in a `fixed-width` face.

{!Html} tags are not escaped automatically;
|1> `the middle <i>words of</i> this line`
are rendered in italic using `\<i\>`.

# Hyperlinks and Anchors

Links can be created by typing

[ displayed ]( target )

where displayed is the text that’s shown for the link and target is the destination.

The target can be anything that the href attribute of an <a> tag can be. For example,

  • it can be an entire URL (like https://site.com),
  • a relative path (like ../page.html), or
  • an anchor on the current page (like #label).

Markdown syntax can be used within displayed (which is how the link above is set in a fixed-width face).

Anchors marking a particular location on the page can be created with

{# label }

where label is the name or id. Only letters, numbers, and the characters _ + - are allowed in label. Although it cannot contain whitespace, there may whitespace before or after.

Anchors themselves are not visible by default (they are empty span elements).

{#label} The start of this paragraph has an anchor, and there’s also
an anchor {# here } at this start of this phrase.

The anchor {# bad anchor } here though is malformed.
The start of this paragraph has an anchor, and there’s also an anchor at this start of this phrase. The anchor {# bad anchor } here though is malformed.

# Variables

A letter or word can be marked as a variable by writing @ before the letter or word. Variable names can only consist of upper- and lowercase letters.

Extra space is selectively placed around delimiters when they are adjacent to variables under the assumption that variables are set in a typical italic face.

For @i = 0, 1, ..., @n\-1,
|1> @x = @inp[@i]
|1> @x = @f(@x)
|1> @out[@i] <- encode[@x],
where @inp is a byte array (of length @n),
@f is an arbitrary function from byte to byte,
and “encode” is a byte array of length 256.

The righthand side of *i* is cramped in [*i*] _vs_ in [@i];
<br>also compare [*j*] _vs_ [@j] and *f*(*x*) _vs_ @f(@x).
For i = 0, 1, ..., n−1, x = inp[i] x = f(x) out[i] ← encode[x], where inp is a byte array (of length n), f is an arbitrary function from byte to byte, and “encode” is a byte array of length 256.

The righthand side of i is cramped in [i] vs in [i];
also compare [j] vs [j] and f(x) vs f(x).

# Superscripts and Subscripts

Superscripts and subscripts are written with

^nnn _nnn

for one or more digits nnn or

^@vvv _@vvv

for some variable vvv or

^{ xxx } _{ xxx }

for any xxx (so long as xxx doesn’t include braces).

Subscripts can be made smaller by doubling the underscore.

@x_1, ..., @x_{@n\-1}, @x_@n are all of type `usize`,
and the maximum value of `usize` is 2^64 &minus; 1.
x1, ..., xn−1, xn are all of type usize, and the maximum value of usize is 264 − 1.

# Abbreviations

There are a few abbreviations provided for convenience:

Text Replacement
>= ≥ greater-than or equal to
<= ≤ less-than or equal to
!= ≠ not equal to
-> → right arrow
<- ← left arrow
\x × multiplication sign
\- − minus sign
-- – en dash
--- — em dash
... ... ellipsis
\~ ... raised ellipsis

When - occurs within a word (specifically, whenever it is not preceded by whitespace), it’s left as U+002D hyphen-minus, since it is assumed to be functioning as a hyphen.

However, when a - is preceded by whitespace

  • and it’s also followed by whitespace, the - is converted to an en dash.
  • and it’s not followed by whitespace, the - is converted to a minus sign.

For example,

Source Rendered
in between - and so in between – and so hyphen converted to en dash
the value is -1 the value is −1 hyphen converted to minus

However, it’s best to use \- or &minus; whenever you intend to write a minus sign, since it’s impossible to determine when to use a minus without semantic information. For example, the syntactic heuristic will completely fail in cases like

Source Rendered
@x-1 x-1 hyphen is not converted to minus sign
@n - 1 n – 1 hyphen converted to en dash rather than minus sign

You should instead write

@x\-1 x−1
@n \- 1 n − 1

or use &minus; or the minus character (U+2212 minus sign) directly.

# Italic and Bold

Text between pairs of _ or pairs of * is rendered in italic. Text between pairs of ** is rendered in bold.

Italic and bold regions can be nested within each other, and italic regions can even be nested in themselves one level if different sigils are used.

This is _in italic with **bold** embedded,_
and **this _still works_ vice-versa**.
You can also *nest italic regions _within_ italic regions*.

However, trying to interleave _italic and
**bold like this_ causes things** to break.
This is in italic with bold embedded, and this still works vice-versa. You can also nest italic regions within italic regions.

However, trying to interleave italic and bold like this_ causes things to break.

Because * is used for italic and bold, you can’t use * for multiplication; use × instead. (That character is U+00D7 multiplication sign, which you can get by writing &times; or \x.) If you need an asterisk for something else, type \*.
